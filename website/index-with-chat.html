How the chat API call works

01
User types a message and hits send. JavaScript builds a messages array: [{ role: "user", content: "What should I eat for lunch?" }]
02
fetch() sends POST to /api/chat on this same domain. credentials: "include" passes basic auth automatically. No API key in the browser. No cross-origin request.
03
NGINX Plus receives /api/chat, checks rate limit (10 req/min), then proxy_passes to 127.0.0.1:3001. Azure OpenAI is never called directly by the browser.
04
Node.js prepends the system prompt (full domain context ~800 tokens), keeps last 10 messages for history, then POST to Azure OpenAI with api-key header.
05
Azure OpenAI responds. Node.js extracts choices[0].message.content and returns { reply: "..." } to the browser. Widget displays it as a bot message.

Request shape (what fetch sends)

Method POST
URL /api/chat (same domain — no CORS)
Header Content-Type: application/json
Credentials include (browser sends basic auth cookie)
Body { "messages": [ { "role": "user", "content": "..." } ] }

Response shape (what Node.js returns)

Success 200 { "reply": "For lunch, try chicken rice..." }
Rate limited 429 Too many requests — NGINX rate limit hit
Unauthorized 401 Basic auth failed or credentials not sent
Server error 500 Node.js error — check journalctl logs
Bad gateway 502 Node.js not running — restart service
